# Bluebirds Secure Assessment Browser (Lockdown Shell)

This directory contains the desktop lockdown browser shell built with **Electron + TypeScript**. It wraps the web-based React/Vite/TanStack Start assessment platform inside a highly secure OS-level shell, preventing cheat vectors.

## 🏗️ Architecture & Security Features

| Feature | Mechanism |
|---|---|
| **Kiosk/Fullscreen Mode** | `kiosk: true`, `alwaysOnTop`, `skipTaskbar` — prevents OS navigation |
| **Anti-Screenshot** | `setContentProtection(true)` — turns screenshots/OBS/Zoom recordings black |
| **Global Shortcut Block** | `globalShortcut.register` — suppresses Alt+Tab, Cmd+Tab, Alt+F4, Ctrl+Alt+Del |
| **Keyboard Intercept** | `before-input-event` — blocks F12, DevTools, Reload inside the renderer |
| **Process Monitor** | `child_process.exec` polling every 3 s — detects blacklisted & VM processes |
| **Clipboard Wiper** | `clipboard.clear()` every 1 s — prevents copy/paste exfiltration |
| **Window Focus Lock** | Re-focuses window on blur events in production |

## 🛠️ Local Development & Setup

### 1. Prerequisites
Ensure you have **Node.js (v18+)** and **npm** installed.

### 2. Install dependencies
```bash
cd desktop-app
npm install
```

### 3. Start the Frontend Dev Server
The Electron shell loads the assessment platform from its URL. In development it points to the Vite dev server:
```bash
cd ../coding-platform-frontend
npm run dev
# Dev server starts at http://localhost:5173 by default
```

### 4. Run the Electron Shell (Development Mode)
Open a new terminal and launch with the `--dev` flag, which:
- Disables kiosk/fullscreen lock (so you won't get trapped)
- Disables content protection (screenshots work)
- Disables clipboard wiping
- Opens Chrome DevTools automatically
```bash
cd desktop-app
npm run start:dev
```

#### Load a custom URL
```bash
APP_URL=http://localhost:3001 npm run start:dev
```

### 5. Run in Production Mode (locally)
```bash
npm start
```
> ⚠️ **Warning:** This applies the full security lockdown. Ensure you can still force-quit via your OS's task manager if needed during testing.

## 📦 Building & Packaging

TypeScript must be compiled before the app can run or be packaged.

| Command | Description |
|---|---|
| `npm run build` | Compile TypeScript → `dist/` |
| `npm run watch` | Compile in watch mode (auto-recompile on save) |
| `npm run dist` | Build + package installers via `electron-builder` |

Packaged installers are output to the `release/` folder:
- **macOS**: `.dmg` (requires Apple Developer ID for notarization in production)
- **Windows**: `.exe` NSIS installer (requires Authenticode signing in production)

## 🔒 Process Blacklist

Processes that trigger a security violation overlay in the assessment UI:

`discord`, `zoom`, `skype`, `teams`, `slack`, `teamviewer`, `anydesk`, `obs`, `obs64`, `snippingtool`, `screenshot`, `whatsapp`, `telegram`, `vncviewer`, `webex`, `gotomeeting`, `chrome-remote-desktop`, `parsec`, `teamviewer_service`

VM indicators that trigger automatic force-submission:

`vboxservice`, `vboxtray`, `vmtoolsd`, `vmware`, `qemu`, `prl_tools_service`, `hyperv`
